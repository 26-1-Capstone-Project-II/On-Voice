# Voice Pitch Guide Bottom Sheet

이 문서는 목소리 시각화 화면 진입 시 노출되는 안내용 바텀시트 UI의
초기 구현 방향을 정리합니다.

## 1. 작업 목적

목소리 시각화 화면에 진입했을 때, 사용자가 스마트폰 위치를 올바르게
맞출 수 있도록 안내하는 바텀시트를 먼저 UI 중심으로 구현합니다.

이번 확장 단계의 목표는 다음과 같습니다.

- 시안 기준의 바텀시트 UI 유지
- 실제 `position.lottie` 애니메이션 삽입
- 진입/닫힘 애니메이션 유지
- 최초 진입 자동 노출
- `!` 버튼을 통한 재오픈 동선 유지
- 바텀시트 오픈 중 상단 버튼 비활성화
- 스와이프 및 배경 탭 dismiss 지원

현재 단계에서도 `다시 보지 않기` 기능과 영속 저장은 범위에서 제외합니다.

## 2. 참고 시안 요약

시안 기준 바텀시트는 다음 요소로 구성됩니다.

- 화면 하단에서 위로 올라오는 바텀시트
- 상단 핸들 바
- `스마트폰 위치 설정` 캡슐형 라벨
- 중앙 안내 문구 2줄
- 중앙 로티 애니메이션 영역
- 하단 `확인` 버튼

중앙 영역에는 `position.lottie` 파일을 실제로 연결합니다.

`다시 보지 않기`는 추후 별도 이슈에서 구현 예정이므로,
이번 단계의 시안 재현 범위에서는 제외합니다.

## 3. 화면 적용 위치

바텀시트는 [FeedbackView.swift](/Users/kdy/CAU/4-1/capstone-II/On-Voice/frontend/OnVoice/View/FeedbackView.swift:11)
에서 제어하는 방향으로 구현합니다.

선정 이유는 다음과 같습니다.

- `!` 버튼이 이미 `FeedbackView`의 네비게이션 바에 존재함
- 최초 진입 자동 노출과 수동 재오픈을 같은 상태값으로 관리할 수 있음
- [VoicePitchView.swift](/Users/kdy/CAU/4-1/capstone-II/On-Voice/frontend/OnVoice/View/VoicePitchView.swift:12)
  는 목소리 시각화 본문 UI에만 집중시키는 편이 구조상 깔끔함

## 4. 구현 방식

SwiftUI 기본 `.sheet` 대신 커스텀 오버레이 바텀시트로 구현합니다.

선택 이유는 다음과 같습니다.

- 아래에서 위로 올라오는 시안 애니메이션을 직접 제어하기 쉬움
- 확인 버튼 클릭 시 아래로 내려가는 닫힘 애니메이션을 정확히 맞추기 쉬움
- 시트가 닫히면서 뒤에 있는 목소리 시각화 레벨 뷰가 자연스럽게 드러남
- 추후 로티, 딤 처리, 재오픈 UX를 유연하게 확장 가능

예상 구조는 다음과 같습니다.

- `FeedbackView`의 루트 `ZStack`
- 배경에 `VoicePitchView`
- 그 위에 딤 레이어
- 최상단에 커스텀 바텀시트 뷰

## 5. 사용자 흐름

기본 진입 흐름은 다음과 같습니다.

1. 사용자가 목소리 시각화 화면(`FeedbackView`)에 진입
2. 자동 표시 조건을 만족하면 바텀시트 노출
3. 바텀시트가 아래에서 위로 등장
4. 사용자가 안내 내용을 확인
5. 사용자가 `확인` 버튼 탭
6. 바텀시트가 아래로 내려가는 애니메이션 실행
7. 애니메이션 종료 후 바텀시트 제거
8. 사용자는 목소리 시각화 레벨 뷰를 사용

재오픈 흐름은 다음과 같습니다.

1. 사용자가 네비게이션 바의 `!` 버튼 탭
2. 안내 바텀시트 다시 표시
3. 동일한 진입/닫힘 애니메이션 재사용

## 6. 상태 설계 방향

초기 UI 구현 단계에서는 아래와 같은 상태 구성을 기준으로 작업합니다.

- `isGuidePresented`
  - 바텀시트 표시 여부
- `isGuideDismissAnimating`
  - 닫힘 애니메이션 진행 여부가 필요할 경우 사용
- `guideSheetDragOffset`
  - 스와이프 dismiss 중 시트의 현재 드래그 위치

현재 단계의 원칙은 다음과 같습니다.

- 화면 진입 시 안내 바텀시트를 자동으로 표시
- `!` 버튼으로는 언제든 다시 표시 가능
- 바텀시트가 떠 있는 동안 `!` 버튼과 `종료` 버튼은 비활성화

주의 사항:

- `다시 보지 않기`와 관련된 상태값은 이번 구현에 포함하지 않음
- 영속 저장이나 조건부 자동 노출은 후속 이슈에서 별도 확장

## 7. 애니메이션 요구사항

기획 의도에 맞춰 아래 동작을 만족해야 합니다.

- 진입 시: 바텀시트가 화면 아래에서 위로 올라옴
- 닫힘 시: `확인` 버튼 클릭 후 바텀시트가 아래로 내려감
- 닫힘 시: 배경 탭 또는 아래 방향 스와이프로도 동일한 dismiss 애니메이션 사용
- 닫힘 애니메이션이 끝난 뒤, 목소리 시각화 화면 사용 상태로 자연스럽게 전환

구현 후보는 다음과 같습니다.

- `offset(y:)` 기반 슬라이드 애니메이션
- `transition(.move(edge: .bottom))`
- 딤 레이어 opacity 애니메이션 동시 적용

실제 구현 시에는 시안 재현성이 높은 방식으로 선택합니다.

## 8. UI 세부 원칙

이번 구현은 시안 우선으로 진행합니다.

- 중앙 로티 영역은 `position.lottie`가 안정적으로 반복 재생되도록 고정 레이아웃 유지
- 컬러, 타이포, 여백은 기존 앱 디자인 토큰에 최대한 맞춤
- `다시 보지 않기` 영역은 이번 단계에서 렌더링하지 않음

## 9. 현재 구현 범위

이번 작업에서 우선 구현할 범위는 다음과 같습니다.

- `FeedbackView`에 안내 바텀시트 오버레이 추가
- 최초 진입 시 자동 노출
- `!` 버튼 탭 시 재오픈
- `확인` 버튼 탭 시 닫힘 애니메이션
- `position.lottie` 애니메이션 표시
- 바텀시트 노출 중 상단 버튼 비활성화
- 바텀시트 아래 방향 스와이프 dismiss
- 딤 배경 탭 dismiss

이번 작업에서 제외하는 범위는 다음과 같습니다.

- `다시 보지 않기` 버튼 UI
- `다시 보지 않기` 상태 및 영속 저장
- 세부 접근성 문구 조정
